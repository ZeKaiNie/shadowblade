# 个人知识库

> 用 Obsidian 打开此文件夹。服务对象：你自己（安全研究知识积累）。
> 项目文档在隔壁 `docs/`，那是给 AI 助手看的。

## 目录结构

```
knowledge/
├── 0_Inbox/       # 原始素材扔这里（安全报告/论文/CVE/博客）
├── 1_Notes/       # AI 帮你编译的结构化笔记（双链互联）
└── 2_MOC/         # 主题地图（≥5篇相关笔记自动生成）
```

## 工作流

1. **投入**：把感兴趣的安全报告、CVE、论文 PDF 扔进 `0_Inbox/`
2. **消化**：让 AI（Claude Code）读取 Inbox 素材，编译成结构化笔记写入 `1_Notes/`
3. **关联**：AI 自动给笔记加双链 `[[]]`，关联已有知识
4. **索引**：当某主题笔记 ≥5 篇，AI 自动在 `2_MOC/` 生成主题地图
5. **Lint**：定期让 AI 扫描孤立笔记、矛盾信息、缺失覆盖

## AI 权限边界

- `0_Inbox/` — AI 可读，AI 消化后可删除/归档已处理素材
- `1_Notes/` — AI 可读写，这是 AI 主要维护区
- `2_MOC/` — AI 可读写，自动维护索引

## 标签规范

笔记 frontmatter 统一用：
```yaml
---
tags: [APT, CVE-2024-xxxx, prompt-injection]
source: "原素材文件名或URL"
date: 2026-04-29
---
```
