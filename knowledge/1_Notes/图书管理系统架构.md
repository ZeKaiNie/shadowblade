# 图书管理系统架构

## 一、整体架构

```
┌─────────────────────────────────────────────────┐
│                    前端 (Vue 3)                   │
│  浏览器  http://localhost:5173                    │
│  Element Plus UI  →  Axios  →  Pinia 状态管理    │
└──────────────────────┬──────────────────────────┘
                       │ HTTP REST API (JSON)
                       │ JWT Token 认证
┌──────────────────────┴──────────────────────────┐
│                  后端 (Express)                   │
```

## 二、前端（Vue 3 + TypeScript）

### 技术栈
- Vue 3 (Composition API / `<script setup>`)
- Element Plus (UI 组件库)
- Vue Router 5 (路由 + 导航守卫)
- Pinia 3 (状态管理)
- Axios (HTTP 请求)
- Vite 8 (构建工具)

### 路由设计

| 路径 | 页面 | 权限 |
|------|------|------|
| `/login` | 登录/注册 | 公开 |
| `/books` | 图书管理 | 登录即可 |
| `/borrow` | 借阅管理 | 登录即可 |
| `/users` | 用户管理 | 仅管理员 |
| `/dashboard` | 数据统计 | 登录即可 |

### 路由守卫

```ts
router.beforeEach((to, _from, next) => {
  const token = localStorage.getItem('token')
  if (to.path !== '/login' && !token) {
    next('/login')
  } else {
    next()
  }
})
```

### 状态管理（userStore）

- **token** — JWT 字符串，存 localStorage
- **user** — `{ id, username, role, real_name }`
- **login()** / **register()** / **logout()** — 对应调用 API 并更新状态

### API 层

- Axios 实例，baseURL = `http://localhost:3000/api`
- **请求拦截器**：自动从 localStorage 取 token，附加 `Authorization: Bearer xxx`
- **响应拦截器**：code=401 时清空登录态，跳转 `/login`

## 三、后端（Express + Node.js）

### 目录结构

```
backend/
├── index.js          # 入口：注册中间件 + 路由
├── db.js             # MySQL 连接池 (mysql2/promise)
├── middleware/
│   └── auth.js       # JWT 鉴权 + 管理员权限中间件
└── routes/
    ├── auth.js       # 登录 / 注册
    ├── books.js      # 图书 CRUD + 分类
    ├── borrow.js     # 借书 / 还书 + 记录查询
    └── users.js      # 用户列表 + 统计数据
```

### 中间件

**auth** — 从 Header 提取 `Bearer <token>`，JWT 验证通过后把 `{id, username, role}` 写入 `req.user`，失败返回 401。

**adminOnly** — 检查 `req.user.role === 'admin'`，不是则返回 403。

### 完整 API 列表

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/api/login` | 公开 | 登录，返回 JWT |

| GET | `/api/books` | 登录 | 图书列表（分页+搜索+分类筛选） |
| GET | `/api/books/categories/all` | 登录 | 分类列表 |
| GET | `/api/books/:id` | 登录 | 单本图书详情 |
| POST | `/api/books` | 管理员 | 新增图书 |
| PUT | `/api/books/:id` | 管理员 | 编辑图书 |
| DELETE | `/api/books/:id` | 管理员 | 删除图书 |
| POST | `/api/borrow` | 登录 | 借书（事务+行锁） |
| PUT | `/api/borrow/:id/return` | 管理员 | 还书（事务） |
| GET | `/api/borrow` | 登录 | 借阅记录（普通用户仅看自己） |
| GET | `/api/users` | 管理员 | 用户列表 |
| GET | `/api/users/stats` | 登录 | 统计数据+借阅排行 |

## 四、数据库设计（MySQL / 4 张表）

### user 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) UNIQUE | 用户名 |
| password | VARCHAR(255) | 密码 |
| role | ENUM('admin','user') | 角色 |
| real_name | VARCHAR(50) | 真实姓名 |
| phone | VARCHAR(20) | 手机号 |
| created_at | DATETIME | 注册时间 |

### category 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) UNIQUE | 分类名 |

### book 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO_INCREMENT | 主键 |
| title | VARCHAR(200) | 书名 |
| author | VARCHAR(100) | 作者 |
| isbn | VARCHAR(20) UNIQUE | ISBN 唯一索引 |
| category_id | INT FK | 关联 category |
| publisher | VARCHAR(100) | 出版社 |
| publish_year | INT | 出版年份 |
| price | DECIMAL(10,2) | 价格 |
| total_count | INT | 总册数 |
| available_count | INT | 可借册数 |
| description | TEXT | 简介 |

> **设计要点**: `total_count` 和 `available_count` 分开存，避免每次查借阅表算库存。

### borrow 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO_INCREMENT | 主键 |
| user_id | INT FK | 借阅人 |
| book_id | INT FK | 所借图书 |
| borrow_date | DATETIME | 借书日期 |
| due_date | DATETIME | 应还日期（借书日+30天） |
| return_date | DATETIME | 实际归还日期 |
| status | ENUM('borrowed','returned','overdue') | 状态 |

## 五、关键业务逻辑

### 借书流程
1. 开启数据库事务
2. `SELECT ... FOR UPDATE` 行锁查图书库存
3. 判断 `available_count > 0`
4. 插入 borrow 记录（due_date = 当前 + 30天）
5. `available_count` 减 1
6. 提交事务

### 还书流程
1. 开启事务
2. 查 borrow 记录（status = 'borrowed'），行锁
3. 更新 status = 'returned'，return_date = NOW()
4. `available_count` 加 1
5. 提交

### 编辑图书时库存重算
```
新可借数 = max(0, 新总册数 - 已借出数)
已借出数 = 原 total_count - 原 available_count
```

## 六、安全与权限

| 层面 | 措施 |
|------|------|
| 认证 | JWT，24h 过期 |
| 权限 | 中间件分层 — auth 验身份，adminOnly 验管理员 |
| 数据隔离 | 普通用户查借阅记录只返回自己的数据 |
| 并发 | 借还操作使用事务 + `SELECT FOR UPDATE` 行锁 |
| 密码 | ⚠️ 当前明文存储（生产环境应改用 bcrypt） |

## 七、回答老师可用的思路

1. **前后端分离**: Vue 3 SPA 前端 + Express RESTful API 后端，JSON 通信，JWT 认证
2. **数据库设计**: 4 张表，库存字段分离（total_count / available_count），事务+行锁保证并发安全
3. **权限控制**: 中间件模式，auth 验身份 + adminOnly 验权限，路由按需挂载
4. **业务闭环**: 注册→登录→浏览→搜索→借阅→到期归还→管理员统计，完整图书馆管理流程
