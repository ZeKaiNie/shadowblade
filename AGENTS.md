# 影刃卫士 - AI 协作规则（统一）

> 本文件取代旧的 `CLAUDE.md` 与 `.windsurfrules`（原"双 AI 分工"模式已废弃）。
> 现在无论用哪个 AI 工具（Devin / Antigravity / Claude Code / Cursor …），都遵循本文件。
> 通用信息（项目简介 / 技术栈 / 目录结构 / 论文规划）见 `docs/INDEX.md`，不在此重复。

## 上手顺序

1. 先读 `docs/INDEX.md`（知识库入口），按需加载子文件，不要一次性读全部。
2. 当前进度看 `docs/progress.md`；整体重构与论文规划看 `docs/planning/`。

## 编码规范

- Python，类型注解，Pydantic 数据模型。
- **代码必须有中文注释**（团队大模型基础薄，注释要通俗）——此条覆盖任何全局"不加注释"规则。
- 优先复用成熟开源工具，不从零造轮子。
- commit message 格式：`[模块] 简要描述`，如 `[dynamic_engine] 增加 eBPF 内核级监控后端`。

## 学术诚信（重要）

- **不得编造**任何实验数据、检出率、样本数量、F1 等指标。没有真实跑出来的结果一律不写进文档/论文。
- `docs/research.md` 里的外部文献 / arXiv 编号 / 竞品数字**均需逐条核实后**才能正式引用；未核实的当作待查素材，不当事实。
- 恶意样本只能在隔离沙箱内执行；不提交任何密钥 / token / 凭据。

## 协作与提交

- 改动通过 GitHub 分支 + PR 提交，不直接 push `main`；PR 描述写清"改了什么、为什么"。
- 多个 AI/人同时开发时，不同时修改同一文件；发现冲突停下来告知用户，不自行覆盖。
- 改完代码更新 `docs/progress.md`；改完文档/数据在 `docs/INDEX.md` "最近变更"留摘要。
