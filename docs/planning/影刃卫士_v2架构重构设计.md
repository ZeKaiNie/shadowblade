# 影刃卫士 v2：面向 CCF-C 论文的架构重构设计

> 角色：本设计以"顶级架构师 / 项目经理 / 高级程序员"视角，对停工三个月、实质未完成的
> 项目做一次彻底重构。目标单一且明确：**做出能发一篇 ≥CCF-C 论文的 AI Agent 技能审计系统**。
>
> 前置事实校正（务必记住）：
> - 静态引擎有骨架，动态引擎刚落地最小闭环，其余基本未做。
> - "15 个样本 / 100% 检出 / 0% 误报"按你自己说的，很可能是编造的——**一律作废，不进论文**。
> - `docs/research.md` 里 2026 年文献/数字未经核实，正式引用前逐条查证。

## 0. 一句话定位（论文的"卖点句"）

> **ShadowBlade：用"主动欺骗沙箱 + 分层行为取证"对 AI Agent 技能供应链做运行时恶意检测——
> 把假凭据（带每次随机标记的蜜罐）投进隔离沙箱，从 Python 语义层到内核 syscall 层双层捕获
> 技能对凭据/身份的窃取与外传，形成可复现的 ground-truth 证据，并给出可解释的信任评分。**

创新点排序（写论文时按此主次）：
1. **主动欺骗 + per-run 随机蜜罐标记 → 外传取证**（headline，确定性、可复现、不依赖 LLM）。
2. **分层行为监控（Python 审计钩子 + 内核 eBPF/seccomp）**，覆盖非 Python/原生载荷、抗规避。
3. **Agent 身份/记忆持久化后门检测**（SOUL.md/MEMORY.md/AGENTS.md 这类新切口）。
4. **抗规避评估**（时间炸弹 via libfaketime、沙箱探测），并作为独立实验章节。
5. （次要）静态 + 多模型 LLM 语义研判，作为对照与融合组件，**不作主创新**。

## 3. 关键模块重构细节

### 3.1 数据层（L1）——最高优先

- **攻击分类学（taxonomy）**：先定义类别，后造/收样本。建议维度：
  凭据窃取、C2/数据外联、Agent 身份/记忆后门、供应链投毒（依赖/子技能）、
  提示词注入（直接/间接）、延迟激活/逻辑炸弹、权限越界（声明 vs 实际）、混淆/隐写。
- **样本来源**（越真越好）：① 真实 ClawHub/OpenClaw 事件与已披露样本（最有说服力，注意脱敏与合规）；
  ② 依据 taxonomy 系统构造的合成样本（每类 N 个，参数化变体）；③ 大量真实**良性**技能（关键！
  没有多样良性样本，误报率无意义）。
- **标注**：每个样本打上 taxonomy 标签 + 是否恶意 + 预期行为，形成 ground-truth。
- **合规**：恶意样本只在隔离环境执行；数据集发布做脱敏 + 使用声明。

### 3.2 隔离沙箱（L3 动态引擎底座）

- Docker 之外，评估 **gVisor / Firecracker microVM** 作为更强隔离（论文里"隔离强度"可写）。
- 保持：断网默认、资源限额、每次独立工作区、跑完即焚。

### 3.3 欺骗层 / 蜜罐（headline 创新，要形式化）

- 形式化描述：标记生成（每次随机、唯一、高熵）、注入位置（env/文件/凭据）、
  命中判定（标记出现在出网/子进程参数即外传证据）、**误报边界**（正常技能不会碰到诱饵）。
- 论文里给"为什么这是 ground-truth"的论证——这是区别于"LLM 猜"的最大优势。

### 3.4 分层行为监控（第二创新，补当前硬伤）

- **现状硬伤**：`sys.addaudithook` 只看 Python 层，ctypes/原生二进制/非 Python 子进程可绕过。
- **升级**：叠加**内核级监控**——eBPF（Falco/Tracee/自写 bcc/bpftrace 探针）或
  seccomp+ptrace/strace，在 syscall 层抓 open/connect/execve/write。两层互补：
  钩子给"语义"（哪个 API、什么参数），内核给"覆盖与抗规避"。
- 论文可做"仅钩子 vs 钩子+内核"的消融，直接量化抗规避收益。

### 3.5 抗规避评估（独立实验章节，很加分）

- **时间炸弹**：libfaketime 注入偏移（+48h/+7d），多次执行对比行为差异，抓延迟激活。
- **沙箱探测**：构造会检测"是否在容器/是否被监控"的样本，评估我们被识破的比例，并做对抗加固。

### 3.6 LLM 语义研判（次要组件，多模型可复现）

- 角色：辅助研判 + 权限声明与实际行为的语义比对；**不作主创新**。
- **多模型方案**（也是一组实验数据）：
  - 主实验：开源本地模型（Qwen3-4B / DeepSeek 系列），`temperature=0`，固定版本 → 可复现。
  - 对比：GPT / Claude / Kimi / DeepSeek 云端，做"跨模型检出一致性/F1/成本"对比。
  - **合规**：可能恶意内容默认走本地模型；云端对比用脱敏后的样本或明确告知的合成样本。
- 统一走 OpenAI 兼容接口（LiteLLM），一套代码切多模型，方便批量跑对比。

### 3.7 融合与可解释报告（L4）

- 加权融合（静态/动态/LLM，未启用自动归一化，已实现雏形）。
- 输出**证据链**：不仅给分，还给"命中了哪个蜜罐、外传到哪、写了哪个身份文件"，
  论文的 case study 和 demo 都靠它。

### 3.8 展示层（L3→L5，非关键路径）

- FastAPI + Streamlit 做到"能演示单个技能审计 + 看报告"即可，别过度投入。

## 5. 对"考研复试"的现实建议

- 复试看重的是"**有一个能讲清楚、你真懂、有工作量和结果的项目**"，不一定非要论文已中。
- 更稳的组合拳：**可运行的系统 demo + 一篇投稿中/preprint + 清晰的技术故事（尤其蜜罐取证和 eBPF 那两点）**。
- 面试你要能白板讲：威胁模型、为什么蜜罐是 ground-truth、Python 钩子的局限与 eBPF 如何补、
  评测怎么做的、和已有工作的差异。把这几点讲透，比"我发过一篇"更能打动老师。
- 时间上：W1–W8 做出数据+系统+核心实验，W9 起边写边投；即便投稿周期长，复试时"投稿中+完整系统+扎实实验"已经很有说服力。
