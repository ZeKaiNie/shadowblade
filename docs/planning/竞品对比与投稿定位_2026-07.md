- **对我们的直接冲击**：**"运行时/动态检测 skill 恶意"这一核心卖点已被至少 3 篇占据**——
  RSA(2606.11671)、Cloak&Detonate(2607.02357)、Dynamic Malicious Skills(2606.16287)；
  我们想"拉上限"的 **SKILL.md 控制面动态注入攻击 + 防御**也已被 2606.16287 做掉（含内核只读挂载防御）。

## 4. 三个"最像"的深剖 + 我们的区别

**① RSA（2606.11671）——最大威胁。** 它已经做了"运行时探测 + 准备执行上下文触发接口 + 从 trace 打标"，
和我们的 harness 入口触发 + 运行时观测**几乎同构**，且报了 90/88/8 的完整数字。
- 我们的区别（需实测坐实，不得空谈）：(a) **声明能力(allowed-tools)↔运行时观测的一致性核验**这一显式建模，RSA 未强调；
  (b) **per-run 随机 canary/蜜罐污点取证**做**确定性、可解释**的证据链（allow/review/deny 带取证 marker），RSA 是 trace 打标、非蜜罐取证；
  (c) 我们主打 **FPR=0 的"确认层"** 而非"高准确率检测器"——定位不同。
- 风险：若只停在"我们也做运行时"，会被认为是 RSA 的子集。**必须靠"零误报确认 + 级联剪枝 + 分层诚实"三件套差异化。**

**② Cloak and Detonate（2607.02357）——对抗视角。** 它证明静态扫描器可被"保义混淆/自解压打包"绕过，
再用动态检测兜底。**这对我们是利好论据**（"静态会被绕过→需要运行时确认"），但也意味着"动态兜底"已被主张过，引用+区分即可。

**③ PrivacyPeek（2606.00152）——"声明 vs 实际"近邻。** 隐私角度审计 agent 获取阶段"拿了任务不需要的数据"。
我们的"声明能力↔观测行为一致性"是**安全/供应链角度**、且带**确定性决策 + 蜜罐取证**，须显式区分。

## 5. 我们能立住的差异点（投稿卖点，均需真实数据支撑）

1. **零误报确认层 + 级联剪枝（最可发）**：贡献叙事 = "把吵的 LLM/静态扫描器（SkillMutator 报 2–17% 检出、
   MalSkillBench 报最强检测器误报很高）的候选，用我们 **FPR=0 的运行时+蜜罐一致性**做确认/剪枝，
   **在固定召回下把误报压下去多少**"。这是**可部署、可量化**的真贡献，且不依赖"我们召回高"（我们召回真值 0.16，不吹）。
2. **确定性、可解释的取证证据链**：per-run 随机 canary/蜜罐 → 命中即"确凿外传证据" → 人类可审计的 allow/review/deny。
   相对 RSA 的 trace 打标、SkillVetBench 的 LLM-as-Judge 打分，我们主打**确定性 + 可解释 + 不靠 LLM 主观**。
3. **诚实的分层适用边界测量**：在 MalSkillBench 上按 CI/PI/MIXED/控制面(B10–B15) 如实分层，
   量化"运行时确认层在哪层有效、哪层天然无信号"。这**正好接住 MalSkillBench 点名的缺口**（无组合能还原代码-指令关系），
   把"召回 0.16"从弱点转成"运行时层适用边界"的干净科学结论。

> 三条里 **①最有发表价值**（deployment/measurement 型贡献，红海里也吃得开），②③是支撑。

## 6. 投稿定位建议

- **现实定级**：CCF-C（如 measurement/security 工程向）/ 中文核心 / SCI Q3–Q2 是**合理目标**；
  CCF-A/B、SCI Q1 在"单点新意已被占 + 3 人本科小组 + 8GB 卡"约束下**短期不现实**（诚实结论）。
- **论文类型**：不要包装成"新 SOTA 检测器"（打不过 RSA/MalSkillBench 作者组）。
  包装成 **"低误报运行时确认层 + 级联 + 分层适用边界的系统化测量与可部署防御"** 论文。
- **必引 & 必区分清单（漏引=审稿直接拒）**：MalSkillBench、RSA、Cloak&Detonate、Dynamic Malicious Skills、
  SkillVetBench、SkillMutator、Locate-and-Judge、PhantomSkill、POISE、SkillCamo、PrivacyPeek。
- **arXiv preprint 时间线**：arXiv 不能挂空占位，须先有 6–8 页初稿；cs.CR 新作者需一次性 endorsement（找导师/已发 cs.CR 者，几分钟~1 天）；
  提交后人工 moderation 通常 1–2 个工作日上线；上线后可无限更新 v2/v3。**瓶颈是初稿写多快**，以当前进度约 1–2 周可到可投 v1。
  **建议尽早挂 v1 抢时间戳**——本领域一个月就变天。
