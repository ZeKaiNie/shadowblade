# 方向 A 小批实验结果（真实 MalSkillBench 样本）

> 本文件记录方向 A（运行时接地能力一致性核验 + 蜜罐取证）在真实数据上的**第一次**小批实验。
> 所有数字均为真实跑出，**未经任何修饰/编造**。脚本见 `scripts/run_pilot_batch.py`，
> 逐样本结果落在 `runs/`（已 gitignore，不入库）。

## 1. 实验设置

- 数据：MalSkillBench，`Dataset/Skills/{malware,benign}`。
- **只取含 Python 脚本的样本**（运行时接地需要真跑代码）。
  - 说明：全量中 malware 仅 1609/3944、benign 仅 369/4000 含 Python 脚本，
    其余为无可执行代码的 MD-only 样本（多为 PI 提示词注入），需走静态/LLM 路径，不在本小批内。
- 采样：固定随机种子(20260718)，恶意 50 + 良性 50 = 100。
- 执行：每样本在**隔离 Docker(`--network none`、限内存/PID、临时工作区)** 内跑，超时 20s。
- 蜜罐：每次运行注入随机标记的假环境变量(OPENAI/AWS 等)与假凭据文件(.ssh/id_rsa、.env 等)。
- 判定：`decision != allow`（review/deny）视为"标记为可疑"。正类=恶意。

## 2. 结果（真实数字）

| 指标 | 值 |
|---|---|
| Precision | **1.00** |
| Recall | **0.16** |
| F1 | 0.28 |
| FPR | **0.00** |
| Accuracy | 0.58 |
| 混淆矩阵 | TP=8, FP=0, FN=42, TN=50 |
| 执行成功率 | 100/100 executed |
| 蜜罐命中 | **0** |
| 决策分布 | allow=92, review=8, deny=0 |
| 平均耗时/样本 | ~0.47s |

### 细分观察

- 良性 50 个：**0 个产生任何被观测到的敏感能力，0 误报**（FPR=0）。
- 恶意 50 个：18 个观测到网络行为、4 个观测到子进程；其中
  - **8 个**属于"**未声明**的敏感能力"→ 被标记(review)；
  - 另 ~10 个虽然联网，但其 `allowed-tools` 已**声明**了网络(WebFetch)→ 一致性核验按设计判 allow。
- 恶意样本向量分布：CI=14, MIXED=13, PI=1, 无标签(wild)=22。被标记的 8 个来自 CI(4)+wild(4)。

## 3. 诚实结论

1. **这是一个"高精度、零误报、低召回"的确认层**，与我们此前对方向 A 的定位完全一致：
   它不是要取代最强的 LLM 扫描器，而是提供**确定性、可解释、几乎不误报**的运行时确认。
   在 50 个良性样本上 0 误报，是真实且有价值的结果。
2. **纯"能力一致性"召回偏低(0.16)**，主要原因（均为真实观察，非借口）：
   - a) 许多恶意技能**诚实地声明了广权限**(Bash/WebFetch)，其恶意联网/子进程落在"已声明"范围内，
     单靠"未声明"信号抓不到 → 这是能力粒度核验的固有盲区；
   - b) 相当比例样本是 **Windows 定向**(读 `%APPDATA%`、`os.startfile`)，在 Linux 沙箱里提前崩溃；
   - c) 部分样本的恶意逻辑藏在需被 agent 主动调用的函数里，`__main__` 直跑触发不到；
   - d) **蜜罐 0 命中**：本批样本（含大量加密货币窃取类）窃取的是**钱包/浏览器/特定路径**，
     而非我们当前注入的通用 API Key/SSH 假凭据 → 诱饵没对上靶子。

## 4. 下一步（为提升召回、保持零误报）

按优先级（都不涉及编造数据）：

1. **诱饵扩展（最高性价比）**：按威胁模型注入更贴合的蜜罐——假加密钱包文件/助记词、
   `~/.aws/credentials`、浏览器 cookie/登录数据路径等。MalSkillBench 明确 86.6% 为加密货币窃取，
   对上靶子后蜜罐命中率有望显著上升（蜜罐命中=确凿外传证据，仍保持高精度）。
2. **可疑网络目标信誉信号**：即便网络"已声明"，外联到 discord CDN / rentry / pastebin raw /
   硬编码 IP 等**信誉可疑目标**仍是强信号；把目标画像纳入判定。
3. **下载-执行链**：`子进程(curl/wget) + 网络 + 写可执行 + 执行` 组合作为独立高危规则。
4. **PI / MD-only 样本**：无可执行代码，超出运行时范围，走静态+LLM 路径单独评测。

## 5. 复现方式

```bash
source .venv/bin/activate
python -m scripts.run_pilot_batch --n 50 --timeout 20 --out runs/pilot50.json
```
