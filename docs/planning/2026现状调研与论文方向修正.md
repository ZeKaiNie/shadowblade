## 一、先纠正我自己之前的怀疑：ClawHavoc 是**真的**

我此前怀疑 "ClawHub/ClawHavoc/OpenClaw" 可能是 AI 编造——**错了，它是真实事件**：
- 多家媒体报道：The Hacker News、Cyber Security News、cyberpress.org、Koi Security(首披露)。
- OpenClaw（前身 ClawdBot / Moltbot）的技能市场 ClawHub，2026.01–02 遭 ClawHavoc 供应链投毒，
  约 1,184（一说 341 确认）恶意技能，窃取凭据 / Atomic Stealer / C2。
- ⚠️ 但**具体数字仍需以原始报告为准**再引用；且 ClawHavoc 只是"野外"样本的**一个**来源。

## 二、坏消息：这个赛道 2026 年已经"卷起来了"，我们原来的卖点被别人做了

检索发现**两篇直接撞题**的学术工作，且都已开源数据 + 代码 + baseline：

### 1. USENIX Security 2026（≈CCF-A）
- 标题：*"Do Not Mention This to the User": Detecting and Understanding Malicious Agent Skills in the Wild*
- arXiv: **2602.06547**；作者：Yi Liu, Gelei Deng, Yuekang Li, Leo Yu Zhang 等
- 数据集：**98,380** 个技能（来自 skills.rest + skillsmp.com），4,287 可疑，**157 个行为确认恶意**
- 方法管线：注册表爬虫 → 静态规则扫描 → **Docker 沙箱行为验证（strace 系统调用 + network.pcap
  抓包 + 文件系统监控 + nova-tracer）** → 可选 LLM(Claude Code) 研判
- 已开源：GitHub `protectskills/MaliciousAgentSkillsBench` + HuggingFace + Zenodo（MIT）

### 2. MalSkillBench（arXiv 2606.07131，2026-06）
- 标题：*MalSkillBench: A Runtime-Verified Benchmark of Malicious Agent Skills*
- 作者含 **Yong Fang（四川大学）**、Yang Liu 等
- **3,944** 个恶意技能 + 108 格三维分类学 + 703 野外 + 4,000 良性；生成-验证-反馈闭环，
  **Docker 沙箱 + 系统调用监控 + LLM judge** 做运行时验证
- 已开源数据 + 管线 + baseline + 结果

**这意味着**：我们最初"唯一做动态沙箱 + 监控"的差异化说法**已经不成立**——别人不但做了，
还用的是比我们现在更强的**内核级 strace/syscall + 抓包**监控（我之前建议补的 eBPF/内核级，
在这两篇里已是标配）。原样往下做 = 重复劳动，投稿必被判 incremental。

### 3. 工具侧也有了
- **NVIDIA SkillSpector**（GitHub）：AI agent skills 安全扫描器
- **safedep** 发布 *Agent Skills Threat Model*
- 近期新闻："Claude Flaw Automatically Sends Malicious Prompts to AI Agents"(Dark Reading)、
  "Top 6 Claude Cowork Security Risks"(CSA) —— 说明这是当前热点，关注者多、门槛高。

## 三、好消息：你"手上什么都没有"的问题，直接被这些公开资源解决了

1. **真实数据集现成可用、可引用、可复现**：
   - `protectskills/MaliciousAgentSkillsBench`（HuggingFace + Zenodo，157 确认恶意 + 98,380 快照）
   - MalSkillBench（3,944 恶意 + 4,000 良性 + 108 格分类学 + baseline）
   - **不用再编样本**。合成只用于补充分类学里样本稀薄的类别，并如实标注。
2. **现成 baseline**：这两篇都放了 baseline 结果，我们可以直接对比，省掉自建对照的大工程。
3. **现成分类学**：可直接对齐/复用它们的攻击分类，避免自造一套没人认的分类。

## 四、更好的消息：它们自己点明了"还没解决的硬问题"——这就是我们的机会

MalSkillBench 原文结论（可直接作为我们选题依据）：
- "最强检测器在**代码注入**上 recall 98.4%，但在**提示词注入（prompt-injection）和
  智能体控制面（agent-control）攻击**上**崩掉**"；
- "供应链扫描器和提示词注入防御**各自只看到技能的一半**，没有任何组合能恢复
  代码-指令的关联关系"；
- "检测恶意技能必须**联合推理 任务意图 + 代码 + 指令**"。

USENIX 那篇的标题本身（"Do Not Mention This to the User"）就是**指令层/提示词层**的攻击。

**=> 明确的开放问题（我们真正能立住的方向，三选一或组合）：**

- **方向 A（最推荐）**：专攻现有检测器**崩掉**的 prompt-injection / agent-control 攻击，
  提出"意图×代码×指令"联合推理的检测方法，在 MalSkillBench 的**难子集**上超过已发布 baseline。
  数据、baseline、分类学全用现成的，工作量集中在"方法 + 实验"，最契合 CCF-C 的可行性。
- **方向 B（我们的老本行差异化）**：把**蜜罐主动欺骗 + per-run 随机标记外传取证**作为
  一个**高精度确认层/降误报层**——现有两篇用 syscall+LLM judge，都**没有用欺骗/蜜罐**拿
  外传的 ground-truth。把它做成"在它们的管线之上，用蜜罐把'疑似外传'确证为'确凿外传'，
  显著降低误报 / 提供可解释证据"，在公开数据集上量化提升。
- **方向 C**：agent 身份/记忆控制面后门（SOUL.md/MEMORY.md 持久化）这个"架构性新尾巴"
  （MalSkillBench 说野外样本里有"小但架构性全新的攻击控制面的尾部"）——做专门检测 + 小数据集。

## 五、对三个具体问题的直接回答

**Q：论文最好用真实样本吗？**
是。而且现在有**真实、公开、可引用**的（上面两个数据集），优先用它们；纯合成会被质疑。
必要时用合成补稀薄类别，务必标注"合成 + 依据的真实攻击手法出处"。

**Q：你能帮我搜集到吗？**
能——直接从 HuggingFace / Zenodo / GitHub 拉公开数据集（合法、带许可证）。恶意样本只在
隔离沙箱内使用，不外传、不入库明文。**注意**：这些仓库明确写了"动态执行只是插桩、不是强隔离
边界，务必用一次性 VM/容器"，我们沿用并加固。

**Q：ClawHavoc 那 1000 多个样本过时吗？背景要更新吗？**
- 背景**必须更新**：从"我们首创动态检测"改为"在 USENIX'26 / MalSkillBench 已确立的
  运行时检测范式之上，攻克它们未解决的 prompt-injection / agent-control / 联合推理问题"。
- ClawHavoc 样本**不是不能用，但不能只靠它**：MalSkillBench 明确指出野外数据"很窄，
  86.6% 是同一种加密货币窃取行为、81% 来自两个账号"——只用 ClawHavoc 会**严重偏置**，
  结论泛化性会被审稿人打。正确做法：以公开基准数据集为主，ClawHavoc 作为一个野外案例补充。

## 七、深挖竞品（第二轮，2026-07 联网）——残酷现实：几乎每个相邻方向都被占了

用户担心"又选到别人做过的方向"。我又逐一核实了所有明显的相邻方向，结论要有心理准备：
**这个赛道已经极度饱和、且以"周"为节奏在更新，主力是 NTU / 港科大(Shing-Chi Cheung、Shuai Wang) /
四川大学 等强组。** 逐条：

| 相邻方向 | 是否已被占 | 代表工作（已核实，2026） |
|---|---|---|
| 静态+动态+LLM 检测恶意技能 | ✅ 已占 | USENIX Sec'26《Do Not Mention…》(2602.06547) |
| 运行时验证基准 + 攻击分类学 | ✅ 已占 | MalSkillBench (2606.07131) |
| 记忆投毒 攻击/防御/取证 | ✅ 已占（拥挤） | 2601.05504、2606.30566(取证)、eTAMP 2604.02623 等多篇 |
| 扫描器规避 / 对抗样本 | ✅ 已占 | Cloak and Detonate (2607.02357, 港科大)、SkillCloak(CSA) |
| **声明能力 vs 实际能力 一致性核验** | ✅ 已占 | **BIV: Behavioral Integrity Verification (2605.11770)** |
| MCP/工具 最小权限运行时管控 | 🟡 工业界为主 | Microsoft AGT、OPA、Sandbox0、capability tokens（多为博客/工程） |

**BIV(2605.11770) 尤其关键**——它就是我之前想推荐的"声明 vs 实际能力"框架：
静态代码分析 + LLM 抽取能力做"声明 vs 实际"的类型化比对，49,943 个 OpenClaw 技能，
906 样本基准上 F1 0.946。**但它的"实际能力"是静态'预测'出来的，不是运行时真实'观测'到的，
也不做蜜罐、不做运行时执行、不做拦截/防御。**

## 八、还剩下的真实缝隙（BIV / MalSkillBench / USENIX 都没同时做的）

把已发表工作叠起来看，还没有人**同时**做到这四点：
1. **运行时真实观测**的"声明 vs 实际"一致性（BIV 是静态预测；我们用动态沙箱观测真实行为）；
2. **蜜罐 per-run 随机标记**把"疑似外传"确证成"确凿外传"的 ground-truth（现有都用 syscall+LLM judge，
   没用欺骗取证）；
3. 专攻现有检测器**崩掉**的 prompt-injection / agent 控制面偏差（MalSkillBench 明说未解）；
4. 从"检测"走到"**拦截/阻断**"（deviation → deny），BIV 只核验不阻断。

**=> 收敛后的选题（唯一我还敢称"有新意"的）**：
> **"运行时接地的技能能力一致性核验与阻断"**——用带随机蜜罐标记的动态沙箱，把技能在
> SKILL.md 里"声明的能力/意图"与"运行时真实观测到的行为（尤其外传/控制面偏差）"做一致性比对，
> 蜜罐命中作为外传的确凿证据，对偏差实施运行时阻断；在公开基准(MalSkillBench /
> MaliciousAgentSkillsBench)上对比 BIV / 扫描器 / 单轮 LLM baseline。

差异化一句话：**"BIV 是静态预测 + 只核验；我们是运行时观测 + 蜜罐取证 + 阻断，专打它们崩掉的
prompt-injection / 控制面外传。"**

## 九、必须对用户讲清楚的三句实话

1. **"足够领先 / 抢先发" 对单人学生队不现实。** 这个方向被 NTU/港科大/川大等强组以"周"为
   节奏刷，你无法在新颖度上跑赢他们、也不可能"抢首发"。硬拼 CCF-A/B 只会挫败。
2. **但 CCF-C / 中文核心 仍然可及**——前提是选一个像第八节那样"窄而实"的缝隙，用公开数据集 +
   公开 baseline，把"运行时接地 + 蜜罐取证 + 阻断"这条别人没合起来做的线做扎实。
3. **时间线要务实**：1 个月 + AI 辅助，能产出"方法 + 公开基准上的实验 + 初稿 + arXiv preprint"；
   但"投出并被 CCF-C 录用"通常还要 3–8 个月审稿。preprint 可以占位、复试可用，但不等于已录用。

**推荐落地目标**：中文核心（如《信息安全学报》《网络与信息安全学报》）为主 + arXiv preprint 占位，
CCF-C 会议为冲刺；CCF-A/B、SCI Q1 不作为目标。
