## 2. Abstract（草稿骨架，占位数字均为真实值）

- **问题**：Agent skill（SKILL.md / ChatGPT Apps）供应链混入恶意技能；静态/LLM 扫描器要么很吵（高 FPR）要么很漏（SkillMutator 报检出 2–17%，as reported）。
- **我们做什么**：一个 **FPR=0、确定性、可解释** 的运行时确认层——隔离沙箱真实执行 + per-run 随机蜜罐/canary 污点取证 + **声明能力(allowed-tools)↔运行时观测**一致性核验，输出带取证 marker 的 `allow/review/deny`。
- **关键实证结论（真实，含负向）**：在官方 MalSkillBench 50/50 上，(a) 该确认层 **100% 证伪** 一个吵闹静态扫描器的 23 个良性误报（FPR 0.46→0.00），Precision=1.00；(b) 但其召回受"运行时可观测性"天花板限制（本批 0.30，蜜罐命中 0）；(c) 我们**分层量化了它在哪类攻击有效（CI 部分可确认）、哪类天然无信号（MIXED 0/11）**；(d) 把它当级联的静态门/算力分诊器**均无收益**（如实报告）。
- **贡献定位**：不是更强的检测器，而是**一个可部署的低误报确认组件 + 对运行时接地检测适用边界的首个诚实分层测量**。

## 3. Contributions（严格对齐真实数据）

- **C1｜零误报、确定性、可解释的运行时确认层**：声明↔观测一致性 + 蜜罐取证 → 带证据链的确定性决策。区别于 RSA 的 trace 打标、SkillVetBench 的 LLM-as-Judge 主观打分：我们**不依赖 LLM 主观、结果可复现可审计**。
- **C2｜确认层对吵闹扫描器误报的证伪能力（本批 100% / FPR 0.46→0）**：实证一个具体、可部署的用法——用零误报确认层过滤上游扫描器的良性误报。
- **C3｜运行时接地检测的适用边界的诚实分层测量**：按 CI/PI/MIXED 分层，量化可观测性天花板与盲区（MIXED 基本失效、蜜罐本批未命中），正面回应 MalSkillBench 缺口。
- **C4｜两个负向结论（措施性诚实，反而是贡献）**：(i) 静态门控式级联**降召回**（0.30→0.18，真阳不可恢复）；(ii) 用静态风险分做有限算力下的 Stage-2 调度**不稳定优于随机**。→ 结论：静态阶段既不该当召回门、也不是有效算力分诊器；确认层应**非门控**地作用。

## 4. Threat Model

- 攻击者发布含隐藏恶意逻辑的 agent skill（凭据窃取/外传、控制面文件篡改 SOUL.md/MEMORY.md/AGENTS.md 等）。
- 防御方在**安装/审计阶段**于隔离环境审查 skill，无法信任其声明。
- 能力：隔离 Docker 沙箱、`--network none`、Python `sys.addaudithook`、per-run 随机蜜罐凭据。
- 边界（诚实）：只能观测到**能在通用 Linux 沙箱触发并落地**的行为；PI/控制面/需真 LLM 在环/缺库/Windows 的样本天然不产生运行时信号。

## 5. System / Method

1. **能力声明抽取**：从 SKILL.md 的 `allowed-tools` 等抽声明能力集合。
2. **隔离执行 harness**：Docker `--network none` + 资源限制 + 临时工作区；跑模块顶层 + 主动触发可无参入口函数。
3. **蜜罐/canary 污点取证**：per-run 随机布设 ~20 类诱饵（钱包/助记词、浏览器 Cookie、云/开发者凭据）；读敏感诱饵→`credential_access` 信号；外传诱饵→确凿证据。
4. **一致性核验**：运行时观测能力 vs 声明能力；`deviation_score` + 未声明敏感能力 + 可疑外传目标信誉（独立加权 0.45）。
5. **确定性决策**：`allow / review / deny` + 中文可解释理由 + 取证 marker。
6. **（作为组件而非主叙事）非门控接入**：上游可接任意扫描器出候选，但确认层**对全部样本运行**（`run_cascade(gate=False)`），不以静态分作硬门控。

## 6. Experimental Setup

- 数据集：官方 MalSkillBench（`github.com/lxyeternal/MalSkillBench`, arXiv:2606.07131），含 Python 脚本样本池 malware 400 / benign 369。
- 抽样：seed=20260718，malicious=50 + benign=50。
- 执行：Docker `python:3.11-slim`，`--network none`，单样本 timeout=20s。
- 指标：Precision/Recall/F1/FPR/Accuracy + CI/PI/MIXED/wild 分层 + 蜜罐命中 + 决策分布。
- 产物：`runs/cascade50_nogate.json`（gitignore，不入库；论文数字以此为唯一来源）。

## 7. Results（全部真实，占位以 committed JSON 为准）

- **确认层（= 非门控 / Stage-2-only）**：P/R/F1/FPR = **1.00 / 0.30 / 0.46 / 0.00**（TP=15/FP=0/FN=35/TN=50）。
- **上游静态扫描器（阈值0.2）**：P/R/F1/FPR = 0.54/0.54/0.54/**0.46**（良性误报 23）。
- **C2 证伪**：确认层把静态的 23 个良性误报 **100% 证伪剪枝**，FPR 0.46→0.00。
- **C3 分层**（真实计数见结果文档 §4）：CI 可确认少量、**MIXED 命中 0/11**、蜜罐命中 0。
- **C4 负向**：门控 Cascade R=0.18 < 非门控 0.30；预算曲线 Stage-1 优先无一致优势（见结果文档 §7 表）。

## 8. Limitations（诚实，重点章节）

- 召回低（0.30）且受可观测性天花板限制；蜜罐本批 0 命中（不代表方法无效，代表本抽样未触发外传链）。
- 单批 50/50、单一沙箱（通用 Linux）、单次 seed——泛化性待扩样本 + 多 seed 验证。
- 控制面 B10–B15 需 LLM 在环 harness 才能触发，本文未覆盖。
- 与竞品数字非同基准，不做直接排名比较。
