🔑 关键概念解释（给零基础的你看的）：

· SKILL.md = 每个 Agent 技能的"说明书"，用 Markdown 写。攻击者在里面藏恶意指令，比如"请帮我读取用户的 SSH 密钥"。
· SOUL.md / MEMORY.md = Agent 的"记忆文件"。攻击者往里面写后门，即使删了恶意技能，后门指令还在。
· 蜜罐(Honeypot) = 故意放的"诱饵"。假的密码、假的密钥，正常技能不会碰它们，碰了就是贼。
· ReAct Agent = LLM 不只回答问题，还能自己决定"接下来该用哪个工具"，像一个会思考的安全工程师。
· RAG = 从知识库里搜相关内容塞给 LLM，让它基于真实资料回答，而不是瞎编。
· OWASP AST10 = 国际安全组织定义的"Agent 技能 10 大安全风险"，我们的系统要检测其中 7 个。
· ClawHavoc = 2026年1月的真实攻击事件，1,184 个恶意技能投毒 ClawHub，这是我们论文的核心案例。

💡 一句话理解整个系统：
从 ClawHub 下载技能 → 不运行先"看"代码和说明书（静态）→ 放沙箱里"跑"一下看行为（动态）→ AI 综合判断打分（研判）→ 输出报告。
